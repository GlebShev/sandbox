## Обзор проекта

Данный проект реализует чат на языке C с использованием TCP-сокетов и вызова `select()` для мультиплексирования ввода-вывода. Проект состоит из двух программ:

- **Сервер (s.c):**  
  Принимает соединения от клиентов, управляет списком подключённых пользователей, обрабатывает сообщения и команды (например, `\users` для получения списка пользователей и `\quit` для выхода), а также корректно завершает работу при получении сигналов (SIGINT, SIGTERM).

- **Клиент (c.c):**  
  Подключается к серверу, позволяет отправлять и получать сообщения в реальном времени и обрабатывает сигналы для корректного завершения работы.

## Функциональные возможности

- **Реальный чат:**  
  Несколько клиентов могут одновременно подключаться к серверу и обмениваться сообщениями. Сервер пересылает каждое сообщение всем участникам.

- **Уникальные ники:**  
  При подключении сервер запрашивает у клиента никнейм, удаляет лишние пробелы и проверяет его на уникальность. Если ник уже используется или пустой, клиенту отправляется сообщение об ошибке, и соединение закрывается.

- **Команды чата:**  
  - `\users` – клиент получает список всех подключённых пользователей.  
  - `\quit <сообщение>` – клиент выходит из чата, сервер оповещает остальных участников о его уходе с прощальным сообщением.
  - Команды проверяются на корректность записи, то есть, например команды `\usersss` не сработают.

- **Обработка длинных сообщений:**  
  Если сообщение превышает размер буфера (без завершающего символа новой строки), сервер отправляет клиенту сообщение об ошибке и игнорирует его.

- **Валидация ввода:**  
  Все входящие строки (ники и сообщения ) обрабатываются функцией удаления лишних пробелов, что предотвращает обработку пустых строк или строк, состоящих только из пробелов.

- **Обработка сигналов:**  
  На стороне сервера и клиента настроена обработка сигналов SIGINT и SIGTERM с использованием `sigaction()`. При получении сигнала сервер рассылает сообщение о завершении работы, а клиент отправляет команду выхода, что позволяет корректно закрыть соединения.

## Как это работает

### Сервер

#### 1. Инициализация сокета и прослушивание
Сервер создаёт TCP-сокет, привязывает его к указанному порту и начинает прослушивание входящих соединений. Для работы с сокетами используются стандартные функции: `socket()`, `bind()` и `listen()`.

#### 2. Мультиплексирование с помощью `select()`
Сервер использует вызов `select()` для отслеживания активности на:
- Слушающем сокете (для новых соединений).
- Клиентских сокетах (для входящих сообщений).

Это позволяет обрабатывать все соединения в одном потоке, не прибегая к использованию новых процессов или нитей(как минимум, это удобно, так как не требуется разделять ресурсы).

#### 3. Управление клиентами и хранение информации о них
Для хранения информации о подключённых клиентах используется связанный список, где каждая запись описывается структурой:

```c
typedef struct Client {
    int socket;           // Сокет клиента
    char nick[NICK];      // Никнейм клиента (фиксированный размер)
    struct Client *next;  // Указатель на следующий элемент списка
} Client;
```

Основные функции для работы со списком клиентов:
- **add_new_cl()** – добавляет нового клиента в список:
  - Выделяет память для структуры.
  - Копирует ник (с обрезкой до NICK-1 символа) и гарантирует корректное завершение строки.
  - Добавляет нового клиента в начало списка.
  
- **del_cl()** – удаляет клиента из списка по его сокету:
  - Проходит по списку, ищет запись с соответствующим сокетом, корректно изменяет ссылки и освобождает память.
  
- **get_nick()** – возвращает никнейм клиента по его сокету, что позволяет при отправке сообщения добавить информацию о том, кто его отправил.

- **uniq_nick()** – функция, которая проверяет уникальность нового никнейма, проходя по всему списку клиентов. Если ник уже используется, функция возвращает 0.

#### 4. Обработка входящих сообщений и команд
- **Обычные сообщения:**  
  При получении сообщения сервер очищает строку от лишних пробелов, добавляет префикс с никнеймом отправителя и рассылает сообщение всем клиентам.
  
- **Команды:**  
  Если сообщение начинается с символа `\`, оно интерпретируется как команда:
  - `\users` – сервер проходит по списку клиентов, собирает никнеймы в одну строку и отправляет её запрашивающему клиенту.
  - `\quit` – сервер рассылает сообщение о выходе клиента, закрывает его соединение и удаляет запись из списка.

#### 5. Обработка длинных сообщений
Если при вызове `recv()` возвращается количество байт, равное размеру буфера (без учёта завершающего символа) и в полученных данных отсутствует символ новой строки, сервер считает, что сообщение слишком длинное. В этом случае клиенту отправляется сообщение об ошибке, и сообщение игнорируется.

#### 6. Обработка сигналов
Обработка сигналов реализована с использованием `sigaction()`:
- Обработчики для SIGINT и SIGTERM устанавливаются с помощью функции `sig_control()`.
- При получении сигнала глобальный флаг (типа `volatile sig_atomic_t`) устанавливается, после чего в основном цикле сервера происходит рассылка сообщения о завершении работы, закрытие всех соединений и корректное завершение программы.

### Клиент

#### 1. Подключение к серверу
Клиент создаёт TCP-сокет и подключается к серверу с использованием IP-адреса и порта, указанных в командной строке.

#### 2. Мультиплексирование ввода-вывода
Клиент также использует `select()` для одновременного отслеживания:
- Сообщений от сервера.
- Ввода пользователя с клавиатуры (STDIN).

Это позволяет клиенту в реальном времени получать сообщения и отправлять свои, не блокируя работу.

#### 3. Обработка сигналов
Как и сервер, клиент устанавливает обработчики сигналов с помощью `sigaction()`. При получении сигнала клиент:
- Устанавливает глобальный флаг завершения.
- Отправляет серверу команду выхода (`\quit`).
- Корректно завершает работу, закрывая сокет.

#### 4. Отправка и прием сообщений
Клиент считывает строки с клавиатуры (с последующим удалением символов новой строки) и отправляет их серверу. При получении сообщений от сервера они выводятся в консоль.

## Сборка и запуск

### Компиляция

Скомпилируйте программы:

```bash
make sc
```

### Запуск сервера

Запустите сервер, указав порт (например, 12345):

```bash
./s 12345
```

### Запуск клиента

Запустите клиента, указав IP-адрес сервера и порт. Например, если сервер запущен на localhost:

```bash
./c 127.0.0.1 12345
```
