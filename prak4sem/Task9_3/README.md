## Общая Идея

Данный проект представляет собой простую систему инвентаря для игровых приложений. Система позволяет добавлять, удалять и использовать различные игровые предметы, такие как оружие, зелья, броня и магическое оружие. Кроме того, реализован итератор для обхода элементов инвентаря.

## Структура Проекта

Проект состоит из следующих файлов:

- **Item.h**  
  Абстрактный базовый класс для всех игровых предметов.  
  **Основные моменты:**
  - Содержит защищённые поля: `name` (название) и `weight` (вес).
  - Объявляет чисто виртуальную функцию `use()`, которую должны реализовать все наследники.
  - Предоставляет геттеры для получения имени и веса.

- **Weapon.h / Weapon.cpp**  
  Класс `Weapon` представляет оружие и наследуется от `Item`.  
  **Особенности:**
  - Дополнительное поле `damage` (урон).
  - Реализация метода `use()`, выводящего сообщение об атаке с указанием урона.
  - Геттер для получения урона.

- **Potion.h / Potion.cpp**  
  Класс `Potion` реализует зелье для восстановления здоровья и также наследуется от `Item`.  
  **Особенности:**
  - Дополнительное поле `healing` (количество восстановления HP).
  - Метод `use()` выводит сообщение о применении зелья.
  - Геттер для получения величины лечения.

- **Armor.h / Armor.cpp**  
  Класс `Armor` представляет броню и наследуется от `Item`.  
  **Особенности:**
  - Дополнительное поле `defense` (уровень защиты).
  - Переопределённый метод `use()`, который выводит сообщение о надевании брони и увеличении защиты.
  - Геттер для получения уровня защиты.

- **MagicItem.h / MagicItem.cpp**  
  Класс `MagicItem` описывает магический эффект.  
  **Особенности:**
  - Хранит строку `magicEffect` с описанием магического эффекта.
  - Метод `castMagic()` выводит сообщение о применении магии.
  - Геттер для получения описания эффекта.
  - Класс служит базой для добавления магических свойств к предметам.

- **MagicWeapon.h / MagicWeapon.cpp**  
  Класс `MagicWeapon` комбинирует свойства оружия и магии посредством множественного наследования от классов `Weapon` и `MagicItem`.  
  **Особенности:**
  - Использует функционал атаки из `Weapon` и магический эффект из `MagicItem`.
  - Переопределяет метод `use()`, объединяя атаку с применением магического эффекта.

- **Inventory.h / Inventory.cpp**  
  Класс `Inventory` реализует контейнер для хранения указателей на объекты типа `Item`.  
  **Основные моменты:**
  - Элементы хранятся в виде динамического массива указателей (`Item**`).
  - Реализована автоматическая динамическая переаллокация массива при добавлении новых предметов.
  - Методы:
    - `addItem(Item* item)` — добавление нового предмета.
    - `removeItem(int index)` — удаление предмета по индексу.
    - `size()` — возвращает количество элементов в инвентаре.
    - `getItem(int index)` — получение предмета по индексу.
    - `contains(const std::string &itemName)` — проверяет наличие предмета с заданным именем.
  - **Итератор:**  
    Вложенный класс `Iterator` перегружает операторы `++`, `!=` и `*`, позволяя обходить элементы инвентаря с помощью методов `begin()` и `end()`. Это упрощает перебор элементов (например, для вывода содержимого или поиска предмета).

- **main.cpp**  
  Основной файл с функцией `main()`, реализующий консольное меню для взаимодействия с пользователем.  
  **Основные функции меню:**
  - Добавление различных типов предметов (оружие, зелья, броня, магическое оружие).
  - Отображение содержимого инвентаря с использованием итераторов.
  - Использование предмета (вызов полиморфного метода `use()`).
  - Удаление предмета по индексу.
  - Проверка наличия предмета в инвентаре по имени.
  - Завершение работы программы.

- **Makefile**  
  Файл сборки, позволяющий скомпилировать проект с помощью команды `make`.

## Кратко про реализацию

- **Полиморфизм:**  
  Абстрактный класс `Item` определяет единый интерфейс для всех игровых предметов, позволяя хранить объекты разных типов в одном инвентаре и вызывать их уникальную реализацию метода `use()`.

- **Множественное Наследование:**  
  Класс `MagicWeapon` наследует от двух классов – `Weapon` и `MagicItem`. Это показывает, как комбинировать различные аспекты (атаку и магический эффект) в одном предмете.

- **Динамическое Управление Памятью:**  
  Класс `Inventory` использует динамический массив для хранения указателей на объекты `Item`. При достижении предела вместимости производится перераспределение памяти для расширения массива.

- **Реализация Итератора:**  
  Вложенный класс `Iterator` в классе `Inventory` позволяет использовать стандартный стиль обхода элементов (аналогичный итераторам STL). Методы `begin()` и `end()` возвращают итераторы для начала и конца коллекции, что облегчает перебор элементов с помощью циклов `for`.

- **Пользовательский Интерфейс:**  
  Функция `main()` реализует консольное меню, позволяющее пользователю создавать, просматривать, использовать и удалять предметы. Это демонстрирует практическое применение полиморфизма и динамического управления памятью.

## Сборка и Запуск

Для сборки проекта можно воспользоваться Makefile. Выполнить в терминале:

```bash
make
./game
```

Для удаления исполняемых файлов выполнить в терминале:

```bash
make clean
```
