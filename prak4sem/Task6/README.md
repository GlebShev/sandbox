# Task6 Шевченко Глеб 212

Данный проект демонстрирует взаимодействие нескольких процессов при помощи системных семафоров System V на базе модели светофора и пешеходов.

## Состав решения

1. **Светофор (light)**  
   Процесс, периодически меняющий состояние семафора, имитирующего светофор:
   - Красный свет: 5 секунд. В этот момент значение семафора = 0. Пешеходы, пытаясь перейти, будут блокироваться.
   - Зелёный свет: 3 секунды. В этот момент значение семафора = 1. Заблокированные пешеходы разблокируются и «переходят дорогу».

   При запуске создаёт или подключается к системному семафору по ключу `SEM_KEY`. В цикле переключает состояние семафора с 0 на 1 и обратно, с заданными временными интервалами.

2. **Пешеход (ped)**  
   Процессы-пешеходы, которые, запустившись с параметром `time_go` (целое число секунд), имитируют следующую деятельность:
   - «Гуляют» некоторое время (`time_go` секунд), затем пытаются перейти дорогу.
   - При попытке перейти они выполняют операцию `semop(..., -1, ...)` на семафоре:
     - Если светофор красный (0), пешеход заблокируется, ожидая зелёного.
     - Если светофор зелёный (1), пешеход сразу может перейти.
   - После перехода пешеход снова «гуляет», повторяя цикл.
   - Все свои действия (ожидание, переход, прогулка) пешеход сообщает в поток stderr, указывая свой PID и статус.

   При удалении семафора, пешеходы при следующем обращении к `semop` получат ошибку и завершат свою работу.

3. **Программа демонтажа светофора (remove)**  
   Данная программа удаляет семафор (вызов `semctl(semid, 0, IPC_RMID)`). После удаления все пешеходы завершают работу, так как попытки вызвать `semop` вернут ошибку.

## Основные файлы

- `com.h` — Заголовочный файл с общим ключом семафора и необходимыми `#include`.
- `light.c` — Исходный код процесса-светофора.
- `ped.c` — Исходный код процесса-пешехода.
- `remove.c` — Исходный код процесса демонтажа семафора.

## Компиляция

Выполните следующие команды в терминале (Linux):

```bash
gcc -o tr light.c
gcc -o ped ped.c
gcc -o rm remove.c
```

## Запуск(пример)

```bash
./tr &
./ped 5 &
./ped 6 &
./ped 7 &
./rm
```
